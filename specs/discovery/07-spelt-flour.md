**תאריך:** 2026-05-29

## Idea

להכניס **קמח כוסמין מלא** ו**קמח כוסמין לבן** כסוגי קמח בעלי שם בבונה המתכון (במקום שדה ה-"אחר" הגנרי), עם המלצות הידרציה/שאור מותאמות; ובמקביל לעצב מחדש את גלריית הפריסטים. תוך כדי הדיון עלתה תובנה רחבה יותר: מנוע הזמנים (Q10) מתאים זמנים לפי טמפ׳ המטבח אבל **מתעלם לגמרי מסוג הקמח** — וזה פער קיים כבר היום עבור מלא/שיפון, לא רק כוסמין.

---

## רקע — איך כוסמין מתנהג

**כוסמין מלא:** גלוטן חלש ופחות אלסטי — בצק דביק ורכרוכי שנקרע בקלות. סופג מים מהר אך לא מחזיק → **הידרציה נמוכה יותר** מחיטה מלאה (לא גבוהה). מתסס מהר. מטופל בעדינות.

**כוסמין לבן:** מנופה, עדין, בהיר. מתנהג **קרוב לחיטה לבנה** עם גלוטן מעט חלש יותר — ידידותי למתחילים.

---

## Decisions

### 1. Data model
מחליפים את `other` בשני שדות עם שם:
```
{ white, wholeWheat, rye, speltWhite, speltWhole }
```
`other` נשאר ב-Zod עם `.default(0)` ל-backward-compat בקריאה בלבד; **לא מוצג ב-UI** (החלטה A מהטיוטה המקורית). מתכונים ישנים (`other: 0`) נקראים תקין.

### 2. UI — בונה הקמחים
5 שורות בעלות שם:

| עברית | שדה |
|---|---|
| לבן | `white` |
| מלא | `wholeWheat` |
| שיפון | `rye` |
| כוסמין לבן | `speltWhite` |
| כוסמין מלא | `speltWhole` |

שדה "אחר" נעלם מה-UI.

### 3. המלצות (`recommendFor`)
כללי כוסמין מתווספים, **ממוקמים לפני** הכלל `white >= 50` (אחרת תערובת 50/50 תיתפס כ"לבן"). כיוון:
- כוסמין מלא (≥30%): הידרציה נמוכה מחיטה מלאה (~76–77%), שאור נמוך יותר (~18) בשל תסיסה מהירה.
- כוסמין לבן (≥50%): קרוב ללבן (~73%), שאור ~18–20.

> המספרים המדויקים ייקבעו ב-Design.

### 4. פריסטים — עיצוב מחדש (מאושר)
7 פריסטים. 3 חדשים, 2 יורדים.

| # | שם | קמחים | מצב |
|---|---|---|---|
| 1 | כפרי קלאסי | 80 לבן / 20 מלא | קיים |
| 2 | כפרי שיפון | 80 לבן / 20 שיפון | **חדש** |
| 3 | לבן בסיסי | 100 לבן | קיים |
| 4 | 70% מלא | 30 לבן / 70 מלא | קיים |
| 5 | 50% שיפון | 50 לבן / 50 שיפון | קיים |
| 6 | 50% כוסמין מלא | 50 לבן / 50 כוסמין מלא | **חדש** |
| 7 | כוסמין לבן רך (מתחילים) | ~60 לבן / 40 כוסמין לבן, הידרציה נמוכה | **חדש** — מחליף את "כפרי קל למתחילים" |
| — | ~~מלא 100%~~ | — | **יורד** |

- "כוסמין לבן רך" הוא ה-on-ramp החדש למתחילים (הידרציה נמוכה = סלחני) — מחליף את הפריסט הגנרי.
- כל פריסט חדש דורש **תמונה** ב-`public/presets/` → 3 asset-ים חדשים.
- מספרי הידרציה/שאור/מלח לכל פריסט ייקבעו ב-Design.

### 5. פיצול הזמנים לפיצ׳ר נפרד (מאושר)
- **Feature 08** = data model + UI + פריסטים + המלצות הידרציה/שאור/מלח. עצמאי, סיכון נמוך.
- **Feature 09** (חדש, follow-up) = **זמנים-מודעי-קמח** — מכפיל מהירות-תסיסה לפי הרכב הקמח על שלב הבולק, כמו ש-Q10 הוא מכפיל לפי טמפ׳. שמרני (≤~15–20% קיצור), נשען על העיגון התצפיתי הקיים ("סיים ב-50–70% תפיחה"). מתקן את הפער הקיים גם למלא/שיפון, לא רק כוסמין. מוגדר ב-Discovery נפרד.
