# Feature: Starter Readiness Gate

## Problem
הבייק כרגע מתחיל מיד אחרי בחירת מתכון, בהנחה שהסטארטר כבר פעיל ובשיא. בפועל, בייקר שפותח את האפליקציה מוכן לאפות — ורק בשלב 1 מגלה שהסטארטר היה במקרר — מאוכזב ותקוע. הפיצ׳ר הזה מוסיף שאלה מפורשת לפני בחירת מתכון: "יש לך סטארטר בשיא?" — ואם לא, האפליקציה תכנן יחד עם הבייקר מתי להאכיל את הסטארטר ומתי לחזור.

## User Story
As a home baker, I want to be asked about my starter's readiness before I dive into recipe setup, so that I never get stuck mid-bake because I forgot to feed my starter.

## Scope — What's In
- שאלת כניסה: "יש לך סטארטר בשיא?" עם הסבר קצר + תמונה ("מה זה שיא?") לפני בחירת מתכון
- מסלול "כן" → ממשיך לבחירת מתכון (ללא שינוי בשאר הזרימה)
- מסלול "לא" → picker תאריך + שעה "מתי הלחם צריך להיות מוכן?"
- חישוב לאחור: זמן מוכנות → ניכוי ~30 שעות בייק → ניכוי 8–10 שעות שיא סטארטר (בסיס 25°C) → **חלון האכלה** (טווח ~2 שעות, לא נקודה בודדת)
- רכיב `TemperatureStepper` הקיים מוצג במסלול "לא" לכוונון הטמפ׳ (default 25°C) — מייד משפיע על החישוב
- כרטיס תוצאה: "האכילו את הסטארטר בין 19:00 ל-21:00 הערב — הוא יהיה בשיא לכ-06:00–08:00 מחר"
- כפתור "הבנתי, אחזור מאוחר יותר" — סיום הסשן
- חסימת זמן בלתי-ריאלי: אם הזמן שנבחר קצר מ~38 שעות מהרגע, להציג המינימום האפשרי

## Out of Scope
- תזכורת / הוספה ליומן (v2)
- חיבור הזמן שנבחר כאן לסשן הבייק שייפתח מאוחר יותר (v2)
- מסלול "האכלתי לאחרונה ב-X" עם ספירה לאחור עד השיא (v2)
- יצירת סטארטר מאפס / החייאת מחמצת (backlog)

## Acceptance Criteria
- [ ] לחיצה על "התחל אפייה" מציגה שאלת הסטארטר לפני כל מסך מתכון
- [ ] "כן" מעביר ישירות לבחירת מתכון (הזרימה הקיימת — ללא רגרסיה)
- [ ] "לא" פותח picker זמן מוכנות
- [ ] picker חוסם בחירת זמן קרוב מ-38 שעות מהרגע ומציג מינימום מוצע
- [ ] הכרטיס מציג טווח האכלה (2 שעות) וטווח שיא (2 שעות), מחושבים מ-25°C בסיס
- [ ] כל טקסט בעברית, פריסה RTL, ללא מחרוזות hard-coded באנגלית בקומפוננטים
- [ ] מבחן ציפה (float test) מוזכר בהסבר ה"מה זה שיא"

## Dependencies
- Depends on: Feature 02 (Bake Session Shell) — נקודת הכניסה "התחל אפייה" שממנה יוצאת השאלה
- Depends on: עיצוב מגירת המתכון (הפיצ׳ר שנוסף לאחרונה) — הגייט מגיע *לפניה*, לא משנה אותה
- Blocks: אין — מסלול "כן" ממשיך לזרימה הקיימת ללא שינוי
