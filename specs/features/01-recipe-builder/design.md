# Design: בונה מתכון (Recipe Builder)

## Screens Affected

| מסך | מטרה | נקודת כניסה |
|---|---|---|
| **מסך הבית (Home)** | מסך-כניסה של האפליקציה. שתי קריאות-לפעולה: ״התחל אפייה״ (ראשי) ו״המתכונים שלי״ (משני). | פתיחת האפליקציה |
| **Stub מצב אפייה** | מסך-עצירה זמני עד 02 ייבנה. כותרת ״מצב אפייה — בקרוב״ + back. **מינימלי בכוונה** — מוחלף ב-02 בזרימת הבייק האמיתית. | לחיצה על ״התחל אפייה״ במסך הבית |
| **רשימת מתכונים** | רשימת מתכונים שמורים + ״+ מתכון חדש״ + swipe-to-delete. | לחיצה על ״המתכונים שלי״ במסך הבית |
| **גלריית Presets** | בחירה ויזואלית של preset או ״התחל מאפס״. | לחיצה על ״+ מתכון חדש״ במסך הרשימה |
| **טופס מתכון** | יצירה ועריכה (אותו מסך, מצב שונה). | בחירת preset, ״התחל מאפס״, או לחיצה על מתכון קיים |
| **דיאלוג אישור מחיקה** | אישור לפני מחיקה. | swipe + ״מחק״, או כפתור מחיקה בטופס |

אין מסך ״צפייה בלבד״ למתכון — לחיצה על מתכון פותחת ישר עריכה. שמירה בלי שינויים = no-op.

### מבנה מסך הבית
```
                  ״כיכר״

           מה אופים היום?

   ┌──────────────────────────┐
   │   🍞  התחל אפייה          │   ← CTA ראשי, גדול
   └──────────────────────────┘

   ┌──────────────────────────┐
   │   📖  המתכונים שלי   (N) │   ← משני, רגיל
   └──────────────────────────┘
```
- כותרת ראשית: שם האפליקציה (״כיכר״)
- כותרת משנית: ״מה אופים היום?״
- קלף ״התחל אפייה״: עליון, בולט יותר ויזואלית (גובה גדול יותר, אקסנט).
- קלף ״המתכונים שלי״: מתחת, רגיל. מציג ספירה דינמית של מתכונים שמורים.
- אין FAB ואין elements נוספים. שני CTAs בלבד.

## User Flow

```
פתיחת אפליקציה
  ↓
[מסך הבית]
  ├─ ״התחל אפייה״ → [Stub מצב אפייה] → back
  └─ ״המתכונים שלי״ → [רשימת מתכונים]
                          ├─ ריק? → EmptyRecipesState + ״+ מתכון חדש״
                          ├─ יש? → רשימה + ״+ מתכון חדש״
                          │
                          ├─ [+ מתכון חדש]
                          │     ↓
                          │  [גלריית Presets]
                          │     • בחירת preset → טופס פתוח עם ערכים מולאים
                          │     • ״התחל מאפס״ → טופס פתוח ריק
                          │     ↓
                          │  [טופס — מצב יצירה]
                          │     • שמור → רשימה (toast הצלחה)
                          │     • ביטול → דיאלוג ״לאבד שינויים?״ אם נגעת
                          │
                          └─ [לחיצה על מתכון קיים]
                                ↓
                             [טופס — מצב עריכה]
                                • שמור → רשימה
                                • ביטול → רשימה
                                • מחק → דיאלוג → רשימה
```

**Swipe-to-delete מתוך הרשימה:**
```
משיכה ימינה (RTL — מימין-שמאלה מבחינת אצבע) על שורת מתכון
  ↓
חשיפה הדרגתית של אזור-מחיקה משמאל (לפי `ui-playbook` §11)
  ↓
חציית סף → toast עם כפתור ״בטל״
  ↓
2400ms ללא ביטול → מחיקה סופית
```

## States

### מסך הבית
- **Loaded** בלבד. שני קלפים תמיד נראים. הספירה ב״המתכונים שלי״ נטענת מ-localStorage. אם 0 מתכונים — הספירה מוסתרת (לא ״(0)״, פשוט בלי).

### Stub מצב אפייה
- **Stub** בלבד. הודעה ״מצב אפייה — בקרוב״ במרכז, וכפתור-חזרה. אין loading, אין error, אין empty.

### מסך רשימה
- **Empty**: איור/אימוג׳י (🍞), כותרת ״עדיין אין מתכונים״, תיאור ״הוסף את המתכון הראשון שלך כדי להתחיל״, כפתור ״+ מתכון חדש״ במרכז
- **Loaded**: רשימה ממוינת לפי `updatedAt` יורד, כפתור ״+ מתכון חדש״ בראש (לא FAB — כפתור inline בכותרת)
- **Loading initial**: לא רלוונטי — קריאה מ-localStorage סינכרונית-מספיק. אם Tech Lead בוחר IndexedDB, skeleton קצר (פחות מ-200ms — אחרת לא להציג).
- **Error**: לא צפוי לאחסון מקומי. אם אחסון נכשל ב-write → toast ״לא הצלחנו לשמור — נסה שוב״.

### גלריית Presets
- **Loaded** בלבד (presets hardcoded). 6 קלפים בגריד 2×3 (mobile portrait) או 3×2 (mobile landscape / רחב יותר), + קישור משני ״התחל מאפס״ למטה.

### מסך טופס
- **Initial-create**: כל השדות ריקים אלא אם נכנסו מ-preset (אז כולם מולאו).
- **Initial-edit**: שדות טעונים מהמתכון הקיים.
- **Editing-dirty**: המשתמש נגע בלפחות שדה אחד; כפתור שמור פעיל (אם תקין); ניווט אחורה מציג ״לאבד שינויים?״
- **Validating**: שגיאות מוצגות בלייב *רק אחרי* המשתמש נגע בשדה (לא בכניסה ראשונה). כפתור שמור disabled אם יש שגיאות.
- **Saving**: 200ms+ של write → כפתור שמור נכנס למצב ״שומר…״ עם פס פעימה. בפועל, צפוי שזה יקרה תוך פחות מ-100ms — אם הזמן < 200ms, לדלג על המצב הזה כדי למנוע פלאש.
- **Saved**: ניווט חזרה לרשימה + toast ״המתכון נשמר״.

### דיאלוג מחיקה
- **Open**: בקשת אישור.
- **Confirmed**: סגירה + toast עם undo.
- **Cancelled**: סגירה ללא שינוי.

## Interaction Specs

### State machine — `HomeCta` (קלף-CTA במסך הבית)
- **Idle**: קלף עם אייקון + label (+ count אם רלוונטי). הקלף הראשי (״התחל אפייה״) גדול יותר ומדגיש accent. המשני רגיל.
- **Press**: scale 0.985 (cards גדולים — לפי playbook §2), bg ink-06, 120ms ease-out
- **Release**: ניווט ליעד המתאים
- אין drag, אין swipe.

### State machine — `RecipeListItem`
```
Idle → Press → (Drag-right?) → Release → Snap/Commit → Idle
```
- **Press (no drag)**: scale 0.965, bg ink-06, 120ms ease-out → release → ניווט לעריכה
- **Drag-right (RTL swipe)**: ברגע שתנועה > 5px, נקה `isPressed`. עוקבים אחרי position.
- **Rubber-band** (לפי playbook §3):
  - 0–120px: 1:1, חשיפת אזור מחיקה אדום משמאל
  - 120–180px: התנגדות 0.3
  - >180px: hard cap
- **Visual commit signal**: ב->= 120px, האדום מעמיק (red-500 → red-600)
- **Velocity-snap** (לפי §3): `v > 0.5px/ms` או `distance > 60px` → snap ל-120px (פתוח); אחרת snap ל-0 (סגור)
- **Spring** (לפי §4): 250ms `cubic-bezier(0.34, 1.56, 0.64, 1)`
- **Commit**: מבצעים מחיקה אופטימית; שורה ״מתקפלת״ (height → 0, 200ms ease-in); toast מופיע עם ״בטל״

### State machine — `PresetCard`
- Press: scale 0.965, bg ink-06, 120ms ease-out
- Release: ניווט לטופס עם ה-preset
- אין drag

### `HintChip` — האינטראקציה המרכזית-החדשה
- **Idle**: מופיע ליד השדה כשערך מומלץ ≠ ערך נוכחי בשדה. טקסט: ״מומלץ: 78% · עדכן״
- **Press**: scale 0.97, 120ms ease-out
- **Tap → Accept**:
  1. השדה ה״רלוונטי״ מקבל אנימציה קצרה (highlight bg 200ms ease-out → ease-in) שמסמנת ״השתנה״
  2. המספר משתנה ל-recommended (לא inputs מצמוץ — שינוי חלק)
  3. ה-Chip נעלם עם fade-out 200ms ease-in
- **Auto-dismiss**: אם המשתמש שינה ידנית את השדה (לפני שלחץ על ה-chip), ה-chip מתעדכן או נעלם — נעלם אם הערך החדש קרוב למומלץ, מעודכן אם רחוק.
- **Touch target**: 44px גובה. הצ׳יפ הוויזואלי יכול להיות 28px גובה, אבל `::before` overlay מורח אותו ל-44 (לפי §10).
- **לעולם לא דורס**: אין auto-fill על שינוי קמחים. רק הצעה.

**הלוגיקה של מה מומלץ** (טבלת lookup, Tech Lead/Engineer יקלוש):
| תנאי | הידרציה | מלח | שאור |
|---|---|---|---|
| ≥80% לבן | 72% | 2.0% | 20% |
| 50-80% לבן | 75% | 2.0% | 20% |
| 50%+ מלא | 80% | 2.2% | 22% |
| 30%+ שיפון | 78% | 2.2% | 25% |
| מעורב אחר | 75% | 2.0% | 20% |

(טבלה ראשונית — Tech Lead יוכל לטייב.)

### State machine — `FlourBreakdownInput`
- 4 שדות מספריים בגריד 2×2 (mobile) או 4 בשורה (רחב)
- **מתחת**: מחוון סכום חי. ״סה״כ: 100% ✓״ (טקסט ירוק עדין) או ״סה״כ: 95% · חסר 5%״ (טקסט אדום עדין)
- ולידציה: blocking על השמירה, לא pop-up. ההודעה גם משמשת כרמז כאשר המשתמש יצר חוסר.

### State machine — `InclusionsSection`
- **Empty (collapsed)**: כותרת ״תוספות (אופציונלי)״ + ״+ הוסף תוספת״ קטן וצנוע
- **Has rows**: כל שורה מוצגת מלאה
- **Add row**: שורה חדשה ריקה slide-in מלמטה, 250ms ease-out, focus אוטומטי על שדה ״שם״
- **Remove row**: לחיצה על אייקון 🗑 (44×44 touch target) → fade-out 200ms ease-in + collapse גובה 200ms ease-in. **אין דיאלוג אישור**, כי הפעולה מכוונת ושוליים-נמוכים.

### State machine — `RecipeFormScreen` (Save button)
- **Disabled**: יש שגיאות ולידציה, או טופס נקי בלי שינויים
- **Enabled-press**: scale 0.97, bg slight darken, 120ms
- **Saving** (אם > 200ms): טקסט הופך ל״שומר…״, אייקון פעימה
- **Saved**: ניווט חזרה + toast (`ui-playbook` §9)

### Animation curves — סיכום
| פעולה | משך | curve |
|---|---|---|
| Press feedback (קלף/שורה/כפתור) | 120ms | ease-out |
| Spring על release אחרי swipe | 250ms | `cubic-bezier(0.34, 1.56, 0.64, 1)` |
| Slide-in (תוספת חדשה, toast) | 250ms | ease-out |
| Fade-out (תוספת שהוסרה, hint שאומץ) | 200ms | ease-in |
| Row collapse על מחיקה | 200ms | ease-in |
| Hint accept → highlight bg | 200ms | ease-out → ease-in |

### Touch targets
- כל אינטראקציה ≥44×44px (לפי §10).
- אייקון מחיקה בשורת תוספת: ויזואל 16-20px, hit area 44×44 דרך `::before` overlay.
- HintChip: ויזואל ~28px גובה, hit area 44px דרך `::before`.

## Optimistic / Sync Notes

לא רלוונטי — אחסון מקומי, single user, אין סנכרון. אבל:
- **שמירה לוקאלית כן צריכה להיות אופטימית במובן ש**: על שמירה מוצלחת, האפליקציה לא מציגה spinner. ניווט מיידי לרשימה + toast. אם השמירה נכשלה (storage full, quota) → toast שגיאה + נשארים בטופס (לא מאבדים את מה שנכתב).
- **מחיקה אופטימית עם undo**: שורה נעלמת מיד; הנתון לא נמחק *באמת* עד שה-toast (2400ms) נסגר. אם משתמש לחץ ״בטל״ → השורה חוזרת. אחרת — מחיקה סופית.

| הידרציה מחוץ לטווח | ״הידרציה צריכה להיות בין 50% ל-100%״ |
| מלח מחוץ לטווח | ״מלח צריך להיות בין 0% ל-5%״ |
| שאור מחוץ לטווח | ״שאור צריך להיות בין 0% ל-40%״ |
| טמפ׳ מחוץ לטווח | ״טמפ׳ צריכה להיות בין 10°C ל-40°C״ |
| שם תוספת ריק | ״שם התוספת לא יכול להיות ריק״ |
| כמות תוספת ≤ 0 | ״כמות התוספת חייבת להיות חיובית״ |

**Toasts ודיאלוגים:**
| מצב | טקסט |
|---|---|
| שמירה הצליחה | ״המתכון נשמר״ |
| שגיאת שמירה | ״לא הצלחנו לשמור — נסה שוב״ |
| מחיקה מהרשימה (toast) | ״\"{שם}\" נמחק״ + כפתור ״בטל״ |
| מחיקה מהטופס (דיאלוג) | כותרת: ״למחוק את \"{שם}\"?״ · תיאור: ״הפעולה לא ניתנת לביטול״ · כפתורים: ״מחק״ / ״ביטול״ |
| לאבד שינויים | ״לבטל את השינויים?״ · ״כן, חזור״ / ״המשך לערוך״ |

### Mirrored elements

- **כל מה שמסמן ״כיוון תקדמות״** מתהפך ב-RTL: chevron-back של כותרת מסך, ה-arrow על FAB, אייקוני ניווט. ב-Lucide יש support אוטומטי דרך `dir="rtl"` ל-`ChevronLeft`/`ChevronRight`; נדרוש שימוש בהם דרך wrapper שמתהפך אוטומטית.
- **Swipe direction**: ב-RTL, swipe ימינה (positive X) מגלה אזור destructive משמאל (לפי `ui-playbook` §11). זה כבר נכון אוטומטית כי הקואורדינטות אבסולוטיות וה-CSS מטפל בהיפוך הוויזואלי.

### Mixed-direction concerns

- **מספרים בתוך טקסט עברי**: שדות עם יחידה (״75%״, ״25°C״) — להשתמש ב-`<span dir="ltr">{number}</span>{unit}` כדי שהפיוז׳ן יציג נכון.
- **שם מתכון** ובאופן כללי כל input חופשי: `dir="auto"` כדי לאפשר עברית/אנגלית במעורב.
- **שדות מספר**: `dir="ltr"` (מספרים תמיד LTR בתוך שדה).
- **״סה״כ״ במחוון הסכום**: צירוף של מילה עברית + מספר + סימן `%`. דוגמה: `<span>סה״כ: </span><span dir="ltr">95%</span><span> · חסר </span><span dir="ltr">5%</span>`.

### Color tokens (סגור ברמת המערכת)
ערכים ב-[`specs/design/tokens.css`](../../design/tokens.css). השמות הרלוונטיים לפיצ׳ר הזה:
- **Surfaces**: `--bg` (cream), `--bg-2`, `--paper` (white), `--line`, `--line-2`
- **Ink**: `--ink` (text primary), `--ink-2` (secondary), `--ink-3` (tertiary)
- **Accent (clay)**: `--accent`, `--accent-2`, `--accent-3`, `--accent-bg` — ה-CTA הראשי במסך הבית (״התחל אפייה״) משתמש ב-`accent`
- **Sage**: `--sage`, `--sage-2`, `--sage-bg` — מחוון ״סה״כ: 100% ✓״ ירוק
- **Status**: `--warn`/`--warn-bg`, `--danger`/`--danger-bg` (אדום למחיקה ולשגיאות ולידציה)

לא צריך להמציא alpha overlays נפרדים: press-feedback מוגדר ע״י `transform: scale(0.965)` מ-`tokens.css`, לא ע״י alpha-tint.
